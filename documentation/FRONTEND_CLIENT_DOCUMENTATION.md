## Project Overview

The MERN-AUTH frontend is a React-based single-page application (SPA) that provides user authentication and email verification features. It uses Vite as a build tool, Redux for state management, Context API for global app data, and Tailwind CSS for styling.

**Key Features:**
- User Registration & Login
- Email Verification with OTP
- Password Reset Flow
- Protected Routes
- Responsive Design
- Toast Notifications

## File Directory Explained

### `/src/App.jsx`
Main application component that sets up routing with React Router. Contains route definitions for all pages (Home, Login, EmailVerify, ResetPassword) and wraps the app with ToastContainer.

### `/src/main.jsx`
Entry point of the application. Sets up:
- Redux Store Provider
- Context API Provider
- Browser Router
- Root ReactDOM render

### `/src/context/appContext.jsx`
Global context for managing:
- `backendUrl` - API base URL from environment
- `isLoggedin` - Boolean flag for login state
- `userData` - User information object
- Methods: `getAuthState()`, `getUserData()`

Functions:
- `getAuthState()` - Checks if user is authenticated on app load
- `getUserData()` - Fetches user data from backend

### `/src/redux/authSlice.js`
Redux slice defining:
- `setCredentials` - Action to save token and user to Redux
- `logout` - Action to clear Redux state

State shape:
```javascript
{
  token: null,
  user: null,
  isAuthenticated: false
}
```

### `/src/redux/store.js`
Redux store configuration combining all reducers (currently only auth).

### `/src/pages/Home.jsx`
Landing page component showing:
- Navbar (with login/logout options)
- Header (personalized greeting)

### `/src/pages/Login.jsx`
Handles both Sign Up and Login:
- Toggle between "Sign Up" and "Login" states
- Form inputs: name (signup only), email, password
- Calls `/api/auth/register` or `/api/auth/login`
- Saves token & user to Redux
- Redirects to home on success

### `/src/pages/EmailVerify.jsx`
Email verification with OTP:
- 6 input fields for OTP entry
- Auto-focus on next field after digit entry
- Paste handling for OTP
- Calls `/api/auth/verify-account`
- Redirects to home on success

### `/src/pages/ResetPassword.jsx`
3-step password reset flow:
1. **Step 1:** Enter email â†’ Calls `/api/auth/send-reset-otp`
2. **Step 2:** Enter 6-digit OTP
3. **Step 3:** Enter new password â†’ Calls `/api/auth/reset-password`

### `/src/components/Navbar.jsx`
Navigation bar with:
- App logo
- Login button (if not logged in)
- User avatar with dropdown menu (if logged in)
  - Verify Email option
  - Logout option

Functions:
- `sendVerificationOtp()` - Sends OTP request
- `logout()` - Clears auth and redirects

### `/src/components/Header.jsx`
Header component showing:
- Profile image
- Personalized greeting with user name
- Welcome message
- Get Started button

### `/src/assets/assets.js`
Central location for importing and exporting all static assets (images, icons).

## API Integration

### Axios Configuration

All API calls use:
- `withCredentials: true` - Sends cookies with requests
- Base URL from environment: `VITE_BACKEND_URL`
- Automatic error handling with Toast notifications

### API Endpoints Called from Frontend

| Endpoint | Method | Purpose | Auth Required |
|----------|--------|---------|---|
| `/api/auth/register` | POST | User registration | No |
| `/api/auth/login` | POST | User login | No |
| `/api/auth/logout` | POST | User logout | Yes |
| `/api/auth/is-auth` | GET | Check authentication | Yes |
| `/api/auth/send-verify-otp` | POST | Send email OTP | Yes |
| `/api/auth/verify-account` | POST | Verify email with OTP | Yes |
| `/api/auth/send-reset-otp` | POST | Send password reset OTP | No |
| `/api/auth/reset-password` | POST | Reset password | Yes |
| `/api/user/data` | GET | Get user information | Yes |

### Request/Response Examples

**Register:**
```javascript
POST /api/auth/register
Request: { name: "John Doe", email: "john@example.com", password: "pass123" }
Response: { success: true, message: "User Registered Successfully" }
```

**Login:**
```javascript
POST /api/auth/login
Request: { email: "john@example.com", password: "pass123" }
Response: { success: true, message: "Login Successful" }
```

**Send Verify OTP:**
```javascript
POST /api/auth/send-verify-otp
Headers: Cookie: token=jwt_token_here
Response: { success: true, message: "OTP sent to your email" }
```

**Verify Email:**
```javascript
POST /api/auth/verify-account
Headers: Cookie: token=jwt_token_here
Request: { otp: "123456" }
Response: { success: true, message: "Email Verified Successfully" }
```

## Common Tasks

### Adding a New Page

1. Create new file in `/src/pages/[PageName].jsx`
2. Add route in `/src/App.jsx`:
   ```jsx
   <Route path="/page-name" element={<PageName />} />
   ```
3. Update navbar or header to link to new page

### Adding a New Component

1. Create new file in `/src/components/[ComponentName].jsx`
2. Import in page and use:
   ```jsx
   import ComponentName from '../components/ComponentName';
   ```

### Using Redux State

```javascript
import { useSelector, useDispatch } from 'react-redux';
import { setCredentials, logout } from '../redux/authSlice';

const Component = () => {
  const { token, user } = useSelector(state => state.auth);
  const dispatch = useDispatch();
  
  const handleLogin = (data) => {
    dispatch(setCredentials(data));
  };
  
  return <div>User: {user?.name}</div>;
};
```

### Using Context API

```javascript
import { useContext } from 'react';
import { AppContent } from '../context/appContext';

const Component = () => {
  const { userData, isLoggedin, getUserData } = useContext(AppContent);
  
  return <div>Name: {userData?.name}</div>;
};
```

### Making API Calls

```javascript
import axios from 'axios';
import { useContext } from 'react';
import { AppContent } from '../context/appContext';

const Component = () => {
  const { backendUrl } = useContext(AppContent);
  
  const fetchData = async () => {
    try {
      axios.defaults.withCredentials = true;
      const { data } = await axios.get(`${backendUrl}/api/user/data`);
      console.log(data);
    } catch (error) {
      console.error(error.message);
    }
  };
  
  return <button onClick={fetchData}>Fetch Data</button>;
};
```

## Troubleshooting

### Token Not Sent with Requests
- Ensure `withCredentials: true` is set in Axios config
- Check that backend CORS allows your frontend URL

### User Data Not Loading
- Verify backend is running on port 4000
- Check token is present in cookies (browser DevTools)
- Verify `VITE_BACKEND_URL` is correct

### OTP Input Not Working
- Check Ref assignment in input fields
- Verify paste handling is working
- Ensure maxLength="1" on inputs

### Email Verification Failing
- Check OTP expiry (24 hours from generation)
- Verify OTP matches exactly (no spaces)
- Check user exists and is not already verified
